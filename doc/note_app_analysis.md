# 📊 노트앱 시장 Pain Point 분석

## 조사 대상
Notion, OneNote, Obsidian + 기타(Evernote, Bear, Apple Notes 등)의 Reddit, 리뷰, 기술 블로그에서 수집한 **실제 사용자 불만** 기반 분석입니다.

---

## 🔥 6대 공통 Pain Point

### 1. 오프라인 = 무력화
| 앱 | 문제 |
|---|---|
| **Notion** | 인터넷 없으면 사실상 사용 불가. 오프라인 모드 매우 제한적 |
| **OneNote** | 클라우드(OneDrive) 강제 의존, 동기화 실패 시 데이터 유실 우려 |
| **Obsidian** | 로컬 파일 기반이라 오프라인 OK, 하지만 동기화(Sync)는 유료($8/mo) |

> **기회**: 로컬 파일 우선(Local-first) + 선택적 클라우드 동기화

---

### 2. 느린 성능 & 무거운 앱
| 앱 | 문제 |
|---|---|
| **Notion** | 대규모 워크스페이스에서 심각한 속도 저하. 모바일 특히 느림 |
| **OneNote** | 동기화 속도 불안정, 대용량 노트북 로딩 지연 |
| **Obsidian** | 플러그인 많아지면 모바일에서 시작 시간 10초+ |

> **기회**: 경량 설계. 앱 실행 2초 이내, 노트 열기 즉시

---

### 3. 데이터 감금 (Lock-in)
| 앱 | 문제 |
|---|---|
| **Notion** | 독자 포맷. 내보내기하면 깨지는 서식, 대량 PDF 내보내기 불가 |
| **OneNote** | .one 독자 포맷. 일반 텍스트 에디터로 열기 불가능 |
| **Obsidian** | ✅ 마크다운 기반으로 가장 개방적 |

> **기회**: **순수 마크다운(.md) + YAML Frontmatter**. 어떤 에디터로든 열 수 있는 진정한 데이터 소유권

---

### 4. 시간 관리와 노트의 단절
| 앱 | 문제 |
|---|---|
| **모든 앱** | 노트를 작성하지만 "언제 할 것인지"는 캘린더 앱에서 따로 관리 |
| **Notion** | 캘린더 뷰가 있지만 타임라인 기반 시간 설계는 불가 |
| **OneNote** | 시간 개념 자체가 없음 |

> **🎯 핵심 차별점**: 노트 = 시간 블록. **타임박싱이 노트의 DNA에 내장**

---

### 5. 복잡한 학습 곡선
| 앱 | 문제 |
|---|---|
| **Notion** | "설정하는 데 시간을 더 쓴다" — 가장 많은 불만 |
| **Obsidian** | 플러그인 설치/설정 없이는 기본기능도 부족하다는 평가 |
| **OneNote** | 인터페이스 직관성 부족, 검색 바가 여러 개 |

> **기회**: Zero-config 철학. 실행 즉시 사용 가능, 설정 없이 즉각 생산적

---

### 6. "빠른 메모"가 빠르지 않다
| 앱 | 문제 |
|---|---|
| **Notion** | 앱 열기 → 페이지 탐색 → 새 페이지 생성 → 로딩... (최소 5~10초) |
| **OneNote** | 유사하게 느리고, 노트북/섹션 구조가 빠른 메모에 불리 |
| **Obsidian** | 모바일에서 볼트 로딩만 수 초 |

> **기회**: **즉시 캡처(Quick Capture)** — 핫키 하나로 어디서든 0.5초 내 메모 시작

---

## 🎯 DailyManager 차별화 전략 제안

```
기존 노트앱:  "무엇을(What)" 기록하는 도구
DailyManager: "언제(When) + 무엇을(What)" 설계하는 도구
```

| 차별점 | 설명 |
|--------|------|
| **⏰ 타임박싱 내장** | 노트 ≠ 문서, 노트 = 시간 블록. 하루의 시간을 설계하며 동시에 기록 |
| **📂 로컬 우선** | 순수 .md 파일 → 데이터 소유권 100% 보장, 오프라인 완벽 지원 |
| **⚡ 즉시 실행** | 앱 열면 "오늘의 타임라인"이 바로 보임. 탐색/설정 불필요 |
| **🪶 경량** | Electron 최소 구성, 불필요한 기능 배제 |
| **🔄 하루 회고** | 계획 vs 실제 비교, 시간 사용 통계 자동 생성 |
| **⌨️ Quick Capture** | 글로벌 핫키로 즉시 메모 → 자동으로 현재 타임블록에 추가 |

## 📋 추가 구현 고려 기능

1. **Pomodoro 타이머 내장** — 타임블록에 집중 타이머 연동
2. **일간/주간/월간 리포트** — 시간 사용 패턴 시각화
3. **태그 시스템** — 블록에 태그 붙여 카테고리별 시간 분석
4. **템플릿** — 반복되는 하루 구조를 템플릿으로 저장/불러오기
5. **드래그앤드롭 이미지** — 붙여넣기만으로 이미지 자동 저장 및 .md 링크 삽입
